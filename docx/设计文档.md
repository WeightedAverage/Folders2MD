## 一、项目概述

### 1.1 项目背景
在日常开发、文档整理、项目管理中，经常需要快速梳理文件夹结构并以可读的文本方式呈现。手动手打目录树耗时且易错，Folders2MD 旨在提供一种**零门槛、快速、美观**的解决方案——只需将文件夹拖入窗口，即可自动生成标准 Markdown 格式的文件夹树，并支持一键复制或保存。

### 1.2 产品定位
- **目标用户**：开发者、技术文档编写者、项目管理人员、内容创作者
- **核心价值**："拖入即出图，复制就走"
- **平台**：Windows（仅需双击 exe 运行，无需安装 Python 环境）

### 1.3 核心功能摘要
| 功能点 | 描述 |
|--------|------|
| 拖拽选择 | 支持从资源管理器直接拖入文件夹到窗口 |
| 按钮选择 | 提供传统"打开文件夹"对话框 |
| 递归扫描 | 生成完整的文件夹树，直达最深一层 |
| Markdown 输出 | 以 `# 📁` 标题 + 代码块包裹的树形文本输出 |
| 预览查看 | 在只读文本框中直接预览生成结果 |
| 一键复制 | 复制全文到系统剪贴板，可粘贴到任意编辑器中 |
| 保存为文件 | 导出为 `.md` 或 `.txt` 文件 |
| 视觉风格 | 深色主题，现代扁平按钮，清晰等宽字体 |

## 三、详细设计

### 3.1 核心算法：树形结构生成

```
函数 generate_tree_lines(root_path, prefix)
  输入：root_path 当前文件夹路径，prefix 缩进前缀
  输出：字符串列表，每一行是一个节点
  流程：
    1. 获取 root_path 下所有条目，按名称排序
    2. 遍历条目:
       a. 拼接路径 path
       b. 确定是否为最后一个条目，选择符号 "└── " 或 "├── "
       c. 将 prefix + 符号 + 文件名 加入结果列表
       d. 如果 path 是文件夹:
          - 递归调用自身，传入新的 prefix (= prefix + "    " 或 "│   ")
          - 将递归结果追加到结果列表
    3. 返回结果列表
```

**设计要点**：
- 使用 `os.listdir()` 而非 `os.scandir()` 保持简单性，可按需排序保证输出稳定。
- 通过 `try/except PermissionError` 处理无权限访问的目录，返回`<无法访问>`避免程序崩溃。
- 可扩展 `ignore_list` 参数过滤隐藏文件或指定文件夹。

### 3.2 Markdown 格式化器

```
函数 format_markdown(root_path)
  输出：完整 Markdown 字符串
  格式：
    # 📁 根文件夹名
    
    ```        (代码块开始)
    根文件夹名
    树形行 1
    树形行 2
    ...
    ```        (代码块结束)
```

**说明**：采用 Markdown 代码块包裹，在任何 Markdown 渲染器中均可保证等宽字体和正确的缩进对齐。

### 3.4 事件交互流程

#### 拖放流程
```
1. 用户从资源管理器拖拽文件夹 -> 鼠标进入窗口
2. dragEnterEvent 触发:
   - 如果 event.mimeData() 包含 URLs，则接受动作
   - 状态栏显示 "松开鼠标以生成目录"
3. 用户松开鼠标 -> dropEvent 触发:
   - 提取第一个 URL，转换为本地文件路径
   - 如果是文件夹，调用 process(path)
4. process() 更新文本域和状态栏
```

#### 按钮点击流程
```
[📂 选择文件夹] -> QFileDialog.getExistingDirectory() -> process()
[📋 复制] -> 获取 TextEdit 文本 -> QApplication.clipboard().setText()
[💾 保存] -> 获取文本 -> QFileDialog.getSaveFileName() -> 写入文件
```

## 四、非功能性需求与考量

### 4.1 性能
- 对于普通项目目录（几百～几千个项目），生成时间 < 0.1 秒。
- 对于极大目录（如系统根目录），可能耗时较久，建议后续版本加入后台线程避免 UI 冻结。

### 4.2 错误处理
- 无权限文件夹：显示 `<无法访问>` 节点，不中断全流程。
- 非法/不存在的路径：状态栏提示错误，不崩溃。
- 未生成内容时点击复制/保存：友好提示。

### 4.3 可用性
- 拖拽区域 = 整个窗口，无死角，符合直觉。
- 按钮带 Emoji 图标，增加辨识度。
- 状态栏及时反馈操作结果（如"已复制"、"已保存"）。
- 窗口默认尺寸 700×600，合理展示内容。

### 4.4 可扩展性
代码结构清晰，预留扩展点：
- `process()` 中可增加选项参数（忽略隐藏文件、仅文件夹、自定义缩进等）
- 过滤逻辑仅需修改 `generate_tree_lines()` 内一行注释代码
- 可增加"导出配置"、"历史记录面板"等模块

## 五、部署与打包

### 5.1 开发环境
- Python 3.8+ 且已安装 PyQt5
- 执行：`python folders2md.py`

### 5.2 打包为单文件 EXE
```bash

- 输出位置：`dist/Folders2MD.exe`
